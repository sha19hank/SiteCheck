# Documentation Rules

- **Detailed Walkthroughs**: When summarizing a development phase or a large refactor (especially those involving hundreds of lines of code or multiple files), the `walkthrough.md` artifact must be extremely detailed. It should exhaustively list architecture decisions, file-by-file breakdowns, data flow changes, typings, and how the changes integrate into the larger system. Do not provide brief high-level summaries for massive code changes; the user wants to understand exactly what was built under the hood.
